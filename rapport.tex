\section{Introduction}

Le numérique occupe aujourd'hui une place centrale dans notre société. Les services web sont devenus indispensables dans de nombreux domaines comme l'éducation, l'administration, le commerce ou la communication. Cependant, cette généralisation du numérique s'accompagne d'un impact environnemental important.

Selon plusieurs études, le numérique représente aujourd'hui environ 4\,\% des émissions mondiales de gaz à effet de serre et près de 10\,\% de la consommation électrique mondiale. Le poids moyen des pages web a fortement augmenté au cours des dernières décennies, entraînant une sollicitation croissante des réseaux, des serveurs et des terminaux utilisateurs.

L'écoconception web consiste à réduire la quantité de ressources nécessaires au fonctionnement d'un service numérique tout en conservant une qualité de service acceptable pour l'utilisateur. Cette démarche repose notamment sur les principes de simplicité, frugalité et pertinence.

Dans le cadre de ce TP, l'objectif est d'analyser l'impact environnemental d'un site web existant, d'identifier ses principales mauvaises pratiques puis de proposer une version écoconçue de la page d'accueil.

\section{Présentation du site étudié}

Le site choisi pour cette étude est le site institutionnel et commercial de Basic-Fit :

\begin{center}
\url{https://www.basic-fit.com}
\end{center}

Plus précisément, l'analyse porte sur la \textbf{page d'accueil} du site.

Ce site a pour objectif de présenter :
\begin{itemize}
\item l'offre commerciale de Basic-Fit ;
\item les abonnements proposés ;
\item les services disponibles ;
\item les clubs et équipements ;
\item différents contenus marketing destinés à convaincre l'utilisateur.
\end{itemize}

Les principaux utilisateurs du site sont les visiteurs souhaitant découvrir l'offre, comparer les abonnements ou s'inscrire.

La page d'accueil constitue la principale porte d'entrée du site et concentre de nombreux contenus visuels, sections promotionnelles et éléments interactifs. C'est donc cette page qui a été retenue pour l'analyse.

\section{Méthodologie}

L'étude a été réalisée en plusieurs étapes :

\begin{itemize}
\item mesure de l'impact environnemental avec EcoIndex ;
\item mesure complémentaire avec WebsiteCarbon ;
\item audit technique via les outils de développement du navigateur ;
\item proposition d'une refonte écoconçue ;
\item réalisation d'un prototype léger en HTML/CSS.
\end{itemize}

L'onglet \texttt{Network} de Chrome DevTools permet d'analyser :
\begin{itemize}
\item le nombre de requêtes HTTP ;
\item le nombre d'images ;
\item les scripts JavaScript chargés ;
\item les polices utilisées ;
\item les fichiers CSS ;
\item la présence de services tiers et de requêtes dynamiques.
\end{itemize}

Les résultats obtenus sont ensuite interprétés au regard des principes d'écoconception web étudiés en cours.

\section{Analyse de l'impact environnemental}

\subsection{Résultats EcoIndex}

L'analyse de la page d'accueil de Basic-Fit avec EcoIndex donne les résultats suivants :

\begin{table}[H]
\centering
\caption{Résultats EcoIndex de la page d'accueil de Basic-Fit}
\begin{tabular}{lc}
\toprule
Indicateur & Valeur \\
\midrule
Score EcoIndex & 17 / 100 \\
Classe & F \\
Poids de page & 6.017 MB \\
Requêtes HTTP & 98 \\
Nombre d'éléments DOM & 3211 \\
\bottomrule
\end{tabular}
\end{table}

Ces résultats montrent que la page présente un impact environnemental important. Le score de 17/100 est particulièrement faible et classe la page dans la catégorie F.

Le poids de la page dépasse 6 MB, ce qui est très élevé pour une page d'accueil. Ce volume de données entraîne une consommation importante de bande passante et d'énergie.

Le nombre de requêtes HTTP est également élevé avec 98 requêtes, ce qui multiplie les échanges entre le navigateur et le serveur.

Enfin, la complexité du DOM est très importante avec plus de 3200 éléments, ce qui augmente fortement le coût de rendu côté navigateur.

\subsection{Résultats WebsiteCarbon}

L'analyse avec WebsiteCarbon attribue au site la note \textbf{B}. L'outil indique que la page est plus propre que \textbf{74\,\% des pages web}.

Ce résultat semble à première vue contradictoire avec le score EcoIndex de F. Cette différence s'explique par le fait que les deux outils utilisent des méthodologies différentes.

WebsiteCarbon se concentre principalement sur les émissions de CO\textsubscript{2} liées au transfert de données et à l'hébergement, tandis qu'EcoIndex prend aussi en compte la complexité du DOM et le nombre de requêtes HTTP. Ainsi, un site peut avoir un score relativement correct sur WebsiteCarbon tout en restant mauvais du point de vue de l'écoconception front-end.

\subsection{Empreinte environnementale estimée}

Selon EcoIndex, pour 1000 visites mensuelles, la page représenterait environ :

\begin{itemize}
\item 39.9 litres d'eau bleue consommés ;
\item 2.66 kgCO2e émis.
\end{itemize}

Ces estimations permettent de mieux visualiser l'impact réel d'une page web lourde et complexe.

\subsection{Interprétation des résultats}

Les résultats montrent que la page d'accueil du site Basic-Fit présente plusieurs problèmes du point de vue de l'écoconception :

\begin{itemize}
\item un poids de page très élevé ;
\item une complexité DOM excessive ;
\item un nombre important de requêtes HTTP.
\end{itemize}

Ces éléments indiquent que la page n'a pas été conçue dans une logique de sobriété numérique, mais plutôt dans une logique marketing privilégiant le visuel et l'expérience utilisateur.

La forte complexité du DOM, avec 3211 éléments, est particulièrement problématique. Elle impacte directement les performances du navigateur et la consommation de ressources côté client.

Le poids élevé de la page suggère également la présence de nombreuses images ou médias lourds, typiques des sites de communication et de promotion commerciale.

\section{Analyse selon les principes d'écoconception}

L'écoconception web repose notamment sur trois principes importants : la simplicité, la frugalité et la pertinence.

\subsection{Simplicité}

La page étudiée présente une structure relativement dense, de nombreuses ressources et une organisation visuelle chargée. Cela va à l'encontre d'une démarche de simplicité.

\subsection{Frugalité}

Le poids élevé de la page, le nombre d'images et le volume de scripts montrent que la page mobilise un grand nombre de ressources. Cette approche n'est pas compatible avec une logique de sobriété numérique.

\subsection{Pertinence}

Certaines fonctionnalités ou ressources chargées paraissent non indispensables pour un site institutionnel ou commercial dont l'objectif principal est de transmettre de l'information utile et d'orienter l'utilisateur vers une offre.

% --------------------------------------------------
\section{Analyse selon le cycle de vie du service numérique}

L'écoconception ne se limite pas au code. Elle doit être prise en compte tout au long du cycle de vie du service numérique.

\subsection{Expression du besoin}

Les besoins principaux des utilisateurs sont relativement simples :

\begin{itemize}
\item découvrir l'offre Basic-Fit ;
\item comparer les abonnements ;
\item consulter les avantages principaux ;
\item trouver un club ;
\item s'inscrire.
\end{itemize}

De nombreuses fonctionnalités observées sur le site actuel ne semblent pas strictement indispensables à ces objectifs.

\subsection{Conception fonctionnelle}

La conception fonctionnelle doit privilégier la clarté et éviter les fonctionnalités superflues. Une page d'accueil peut rester efficace avec peu de sections, à condition que celles-ci soient bien hiérarchisées.

\subsection{Conception technique}

La conception technique de notre proposition repose sur plusieurs choix simples :

\begin{itemize}
\item suppression du JavaScript ;
\item limitation du nombre d'images ;
\item utilisation de polices système ;
\item structure HTML simplifiée.
\end{itemize}

\subsection{Maintenance}

Une architecture simple facilite la maintenance et limite les coûts techniques et énergétiques liés aux futures évolutions du site.

\section{Propositions d'amélioration}

À partir de l'analyse technique réalisée précédemment, plusieurs axes d'amélioration peuvent être proposés afin de réduire l'impact environnemental de la page.

\subsection{Optimisation des images}

\textbf{Problème :} la page contient un nombre élevé d'images, dont certaines sont volumineuses.

\textbf{Solutions proposées :}
\begin{itemize}
\item réduire le nombre d'images non essentielles ;
\item compresser les images ;
\item privilégier les formats modernes comme WebP ou AVIF ;
\item mettre en place du lazy loading.
\end{itemize}

\textbf{Impact attendu :} réduction significative du poids de la page et du temps de chargement.

\subsection{Réduction des scripts JavaScript}

\textbf{Problème :} présence de nombreux scripts JavaScript, notamment liés au tracking et aux animations.

\textbf{Solutions proposées :}
\begin{itemize}
\item supprimer les scripts non essentiels ;
\item limiter les outils de tracking ;
\item regrouper et minifier les fichiers JavaScript ;
\item charger les scripts de manière différée.
\end{itemize}

\textbf{Impact attendu :} diminution de la consommation CPU et amélioration des performances.

\subsection{Optimisation des feuilles de style}

\textbf{Problème :} nombre important de fichiers CSS.

\textbf{Solutions proposées :}
\begin{itemize}
\item regrouper les fichiers CSS ;
\item supprimer les styles inutilisés ;
\item minifier les fichiers.
\end{itemize}

\textbf{Impact attendu :} réduction du nombre de requêtes et amélioration du temps de rendu.

\subsection{Réduction de la complexité du DOM}

\textbf{Problème :} le DOM contient plus de 3200 éléments.

\textbf{Solutions proposées :}
\begin{itemize}
\item simplifier la structure HTML ;
\item supprimer les éléments inutiles ;
\item limiter l'imbrication des composants.
\end{itemize}

\textbf{Impact attendu :} réduction du coût de rendu côté navigateur.

\subsection{Réduction des requêtes externes}

\textbf{Problème :} présence de nombreuses requêtes vers des services tiers.

\textbf{Solutions proposées :}
\begin{itemize}
\item limiter les appels externes ;
\item héberger certaines ressources localement ;
\item supprimer les services non indispensables.
\end{itemize}

\textbf{Impact attendu :} réduction de la latence et amélioration de la sobriété du service.

\subsection{Utilisation de polices système}

\textbf{Problème :} chargement de plusieurs fichiers de polices.

\textbf{Solutions proposées :}
\begin{itemize}
\item utiliser des polices système ;
\item limiter le nombre de variantes.
\end{itemize}

\textbf{Impact attendu :} réduction des requêtes et du poids global.

\section{Réalisation d'une version écoconçue}

\subsection{Objectif}

L'objectif est de proposer une version simplifiée de la page d'accueil du site Basic-Fit respectant les principes d'écoconception web.

\subsection{Choix de conception}

La version proposée repose sur plusieurs choix forts :

\begin{itemize}
\item suppression totale du JavaScript ;
\item utilisation exclusive de HTML et CSS ;
\item absence de bibliothèques externes ;
\item utilisation de polices système ;
\item réduction maximale du nombre d'images.
\end{itemize}

Une seule image principale peut être conservée afin de maintenir une attractivité visuelle cohérente avec l'identité du site.

\subsection{Structure de la page}

La page écoconçue contient les sections suivantes :

\begin{itemize}
\item en-tête avec navigation simplifiée ;
\item section d'introduction (\textit{hero}) ;
\item présentation des abonnements ;
\item avantages principaux ;
\item appel à l'action ;
\item pied de page.
\end{itemize}

\subsection{Justification des choix}

Ces choix permettent de réduire :

\begin{itemize}
\item le poids de la page ;
\item le nombre de requêtes HTTP ;
\item la consommation CPU ;
\item la complexité du DOM.
\end{itemize}

Tout en conservant les informations essentielles pour l'utilisateur.

\subsection{Architecture}

\begin{center}
Utilisateur $\rightarrow$ Navigateur $\rightarrow$ HTML/CSS $\rightarrow$ Serveur
\end{center}

Cette architecture est volontairement simple afin de réduire le nombre de ressources nécessaires.

\section{Démonstration de la solution proposée}

Afin d'illustrer concrètement les principes d'écoconception étudiés dans ce projet, une page d'accueil alternative du site a été développée sous la forme d'un prototype.

Cette démonstration permet de montrer qu'il est possible de concevoir une interface web plus légère tout en conservant les informations essentielles pour l'utilisateur.

\subsection{Structure de la démonstration}

Le prototype réalisé correspond à une page d'accueil simplifiée du site Basic-Fit. Cette page contient les éléments suivants :

\begin{itemize}
\item une section d'introduction présentant l'offre ;
\item une section dédiée aux abonnements ;
\item une section présentant les avantages principaux ;
\item un appel à l'action final ;
\item un pied de page contenant les informations essentielles.
\end{itemize}

La navigation est volontairement simple et directe afin de limiter la complexité de l'interface.

\subsection{Technologies utilisées}

La page a été développée uniquement avec les technologies suivantes :

\begin{itemize}
\item HTML ;
\item CSS.
\end{itemize}

Aucun script JavaScript ni aucune bibliothèque externe n'ont été utilisés. Cette approche permet de réduire le nombre de requêtes HTTP et la consommation de ressources côté navigateur.

\subsection{Exécution du prototype}

Le prototype peut être exécuté localement en ouvrant simplement le fichier \texttt{index.html} dans un navigateur web ou via un petit serveur local.

Cette simplicité d'exécution permet également de présenter facilement la solution lors de la démonstration finale.

\subsection{Bénéfices observés}

La version proposée présente plusieurs avantages :

\begin{itemize}
\item réduction du nombre de ressources chargées ;
\item diminution du poids global de la page ;
\item simplification de la structure HTML ;
\item meilleure lisibilité du contenu.
\end{itemize}

Cette démonstration montre qu'une approche d'écoconception permet de réduire l'impact environnemental d'un site web tout en conservant une interface fonctionnelle et accessible.

\section{Limites de l'étude}

Cette étude présente plusieurs limites :

\begin{itemize}
\item l'analyse a été réalisée uniquement sur la page d'accueil ;
\item les outils de mesure donnent des estimations et non des valeurs absolues exactes ;
\item la version écoconçue proposée constitue une preuve de concept et non un déploiement complet.
\end{itemize}

La version écoconçue réalisée dans ce projet n'a pas été déployée sur un serveur public. Il n'a donc pas été possible de l'évaluer directement avec les outils EcoIndex et WebsiteCarbon, qui nécessitent une URL accessible en ligne.

Néanmoins, la comparaison des caractéristiques techniques permet déjà d'estimer une amélioration significative en matière de sobriété numérique.

\section{Conclusion}

L'étude montre que la page d'accueil actuelle de Basic-Fit présente plusieurs inefficacités environnementales : poids important, nombreuses images, nombreux scripts, services tiers et structure DOM complexe.

La version écoconçue proposée démontre qu'il est possible de concevoir une interface plus simple, plus légère et plus respectueuse de l'environnement tout en conservant les informations essentielles.

Ce travail met en évidence l'importance des choix réalisés dès la conception, et montre qu'une approche fondée sur la simplicité, la frugalité et la pertinence permet d'améliorer significativement la sobriété d'un service web.
